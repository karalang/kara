Fathom

A Mandelbrot explorer in one Kāra source — the per-frame compute fans out across every CPU core, in a browser tab, with no async and no threading code. Scroll to zoom toward the cursor; drag to pan; double-click to dive IN, right-click to back OUT; arrow keys / + - / R from the keyboard. Click away and rendering pauses; click back and it resumes — an unfocused tab idles the worker pool instead of burning it.

Where is the threading code? There isn't any — the compiler found it. The render loop is a plain blocking loop { frames.recv(); /* drain wheel/pointer/keys/clicks/focus */ if !paused { render_frame(); } }; each frame's rows are split across a Web Worker pool by TaskGroup.spawn, and live scroll/pointer input arrives as channels drained with try_recv — no await.
