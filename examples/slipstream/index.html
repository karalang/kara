Slipstream

A real-time Lattice-Boltzmann wind tunnel in one Kāra source. Air streams left-to-right past the tilted wing; the colour is vorticity, so the von Kármán wake lights up red/blue. The simulation grid is state carried and evolved across frames — the new thing on the browser spine. Drag the slider (or ↑ / ↓ / scroll) to rotate the wing; push past the stall angle and the flow separates. R resets.

The render loop is a plain blocking loop { frames.recv(); /* advance fluid */ put_pixels(); } — no async, no await. Each frame advances the D2Q9 solver a few substeps from the previous frame's grid and blits the vorticity field. Each substep's collide + stream passes fan out across the Web Worker pool via TaskGroup.spawn, every worker reading the one shared grid — yet there is no threading code, no locks, no SharedArrayBuffer juggling in the Kāra source. Where is it? The compiler put it there.
