- [ ] Type system: add `f16` (IEEE 754-2008 half-precision) and `bf16` (bfloat16) as primitive types with the same trait surface as `f32`/`f64` (`PartialEq`, `PartialOrd`, arithmetic traits, `Copy`) but NOT `Eq`/`Ord`/`Hash`.
- [ ] Codegen: lower `f16` → LLVM `half`, `bf16` → LLVM `bfloat`. Native instruction emission on capable targets; software promotion to `f32` on others with a `f16_software_emulated` performance lint.
- [ ] Implicit widening: `f16` → `f32`, `bf16` → `f32` (both lossless).
- [ ] Literal suffixes: `1.0f16`, `1.0bf16`.
- [ ] Stdlib: `F16`, `BF16` total-order wrappers (same pattern as `F32`/`F64`).
- [ ] `Tensor[f16, Shape]` and `Tensor[bf16, Shape]` valid once both this and the numerical stdlib ship.

Typechecker `infer_tensor_from` + `collect_tensor_literal` (`src/typechecker/expr_call.rs`): leftmost spine establishes dims; sibling levels checked against it — length raggedness, depth raggedness (both directions), and scalar/nested mixing are compile errors with the offending sub-literal's span; empty levels rejected (zero-size tensors go through `zeros`); element type from the first leaf, remaining leaves checked against it; leaves are arbitrary expressions.   - [x] **`iter_axis(n)`** axis iteration (2026-06-06). `t.iter_axis(n)` yields the `dims[n]` sub-tensors obtained by fixing the index along axis `n` — the axis is *dropped* (NumPy `take(i, axis=n)` semantics) — as a `Vec` of **copies** (directly `for`-iterable / indexable / `len()`-able; honest about eager-copy semantics since the interpreter has no view/stride machinery, and a lazy view type is v1.5+). Surface decl in `tensor.kara` is doc-only. 9 typechecker + 5 interpreter tests (rows/cols, rank-3 middle axis, copies-not-views, runtime axis + OOB, E_SHAPE-style item-dim mismatch, dim-unified signature flow, `?`-dim survival).
  - [x] **reshape / permute / slice / squeeze** (2026-06-06). Whole family in one slice — typing in the new `src/typechecker/expr_method_tensor.rs` module (`infer_tensor_shape_method` entry; `infer_tensor_iter_axis` moved there too, so all five shape-transform intercepts live together), interpreter twins in `method_call_tensor.rs`, doc-only decls in `tensor.kara`. **Shared posture**: receiver rank must be statically known (bare-`S` / splice → focused error, parameterized per method); literal arguments give exact static dims, runtime-valued arguments degrade the affected dims to `?` (the partially-dynamic posture); all four return *copies* (tensors are value types; codegen may share buffers copy-on-write later); every compile-time check re-emits at runtime (run_program bypass). **reshape(dims-literal)**: arg must be an array literal (its length = result's static rank — a runtime Vec can't provide one; runtime-shaped reshape is v1.5); integer-literal entries → concrete dims, expression entries → `?`; element-count product checked at compile time when receiver + new dims fully static, at runtime always; data buffer unchanged (C-order reinterpret, copied). **permute(axes-literal)**: array literal of *integer literals* forming an exact permutation of 0..rank, fully compile-checked (repeat / OOB / wrong-length / non-literal each get focused errors — runtime perms are v1.5); result dim `i` = receiver dim `perm[i]` (NumPy `transpose(axes)` semantics — pinned by rank-3 `[2,0,1]` test); `?` dims move with their slot; interpreter reorders via output-odometer × source strides. **slice(axis, start, end)**: contiguous `[start, end)` band along one axis, other axes untouched (`t[i, :, :]` syntax is v1.5); literal axis keeps all dims except the sliced slot (= `end−start` when both bounds literal, else `?`); runtime axis → rank-preserved all-`?`; compile range check when dim concrete + bounds literal, runtime check always; `start == end` legal (dim 0); copy walks outer × band × inner. **squeeze(n) / squeeze()**: `squeeze(n)` drops slot `n` (size must be 1 — compile error when concrete non-1, runtime check when `?`; rank ≥ 2 since rank-0 isn't expressible; runtime `n` → rank−1 all-`?`); `squeeze()` requires a fully-static shape (a `?` dim's size — hence result rank — isn't compile-known; error points at `squeeze(n)`), drops all 1-dims, all-ones shape rejected (rank-0), no-1s shape is a legal no-op. 14 typechecker + 6 interpreter tests (incl. chained permute→reshape pinning real reordered copies, copies-not-views, runtime product/bounds/size-1 re-checks).

### Embedded / Hardware Primitives

> Companion work: the embedded *target* itself (Cortex-M, vector tables, linker scripts) is Phase 10 — see [`phase-10-targets.md`](phase-10-targets.md); these are the language/stdlib primitives that co-design with it. Already shipped while this section was parked in phase-8: linker control attributes (`#[link_section]` / `#[no_mangle]` / `#[used]`, ✓ 2026-04-28) — entry stays in `phase-8-stdlib-floor.md § Pre-existing Phase 8 work`.

- [ ] **Volatile memory access (MMIO).** Implement `volatile_read[T: Copy]` and `volatile_write[T: Copy]` as `unsafe` intrinsics. Implement `VolatileCell[T: Copy]` stdlib wrapper. Effect integration: `volatile_read` → `reads(Hardware)`, `volatile_write` → `writes(Hardware)`.

- [ ] **Inline assembly (`asm` / `global_asm`).** Implement `asm` as a keyword expression inside `unsafe` blocks with the LLVM operand model. Per design.md § Inline Assembly (v60 item 21). Operand forms: `in(constraint) expr`, `out(constraint) binding | _`, `lateout(constraint) binding`, `inout(constraint) lvalue` and `inout(constraint) expr => binding`, `inlateout(...)`, `const expr` (compile-time integer substituted into the template), `sym path` (Kāra function/static linker symbol), and the ABI clobber form `clobber_abi("C", "system")`. Constraints accept `reg`, architecture-specific register classes (`reg_byte`, `freg`, `vreg`, ...), and named registers (`"eax"`, `"r0"`, ...). Options: `nomem`, `nostack`, `preserves_flags`, `pure`, `readonly`, `volatile`, `att_syntax`, and `raw` (disables `{...}` template substitution; no operands allowed when set). Effect integration: default → `reads(Hardware) + writes(Hardware)`; `options(nomem, nostack)` → no hardware effects; `options(pure, nomem, nostack)` → no effects. The `clobber_abi(...)` form expands at codegen to the per-target caller-saved register set so call instructions inside asm don't require manual register enumeration. `sym` operands lower to LLVM `s` constraints with the resolved linker symbol; `const` operands lower to the LLVM `i` constraint with the const-evaluated integer. `global_asm` as a top-level item with no operands; symbols defined in `global_asm` are referenced from Kāra via `extern` declarations and inherit the existing FFI effect annotation surface. Deferred (P1+/P2): `label(name) { ... }` blocks (asm goto — design pinned in design.md § Deferred Forms); `#[naked]` functions with `naked_asm` (design pinned in design.md § Deferred Forms — needs coherent integration with effect inference, unsafe-fn, and fn-pointer typing). Test plan when the implementation lands: per-target register-class round-trips, `clobber_abi("C")` round-trip with a stub `extern "C"` callee, `sym` reference to a Kāra `fn` with the right linker name, `const` substitution with arithmetic constant expressions, `raw` template with literal `{ }`, options-driven effect-set computation. (P1 — design surface lands at v1 per v60 item 21; implementation post-v1.)

- [ ] **`Atomic[T]` and memory ordering.** Implement `Atomic[T]` as a `shared struct` with `load`, `store`, `swap`, `compare_exchange`, `fetch_add`, `fetch_and`, `fetch_or` methods. `T` must be `Copy`. Implement `Ordering` enum (`Relaxed`, `Acquire`, `Release`, `AcqRel`, `SeqCst`). Implement `compiler_fence` (safe) and `fence` (unsafe). Available in `embedded` profile for ISR-to-main communication. **Effect-checker integration (design.md § Effect Model, under Atomic Operations):** every atomic method call (`load`, `store`, `swap`, `compare_exchange`, `fetch_add`, `fetch_and`, `fetch_or`) must contribute zero effects to the enclosing function's inferred set — the checker's transfer function treats them as no-ops for effect purposes. Memory-ordering arguments are discarded at the effect layer and forwarded to codegen. `lock` blocks are effect-transparent: their body's effects flow to the enclosing function unchanged; the lock itself contributes nothing. **Auto-concurrency integration (Phase 6):** treat atomic-only function bodies as conflict-free — two such functions may run in parallel on the same atomic value, because atomic ops are linearizable. The Phase 2 `Rc → Arc` promotion analysis handles atomic fields inside a `shared struct` uniformly with non-atomic fields — no special case needed.

- [ ] **`#[interrupt]` handler ABI.** Implement `#[interrupt(NAME)]` attribute: forces interrupt calling convention, emits vector table placement, forbids direct calls from normal code, implicitly applies ISR profile restrictions (no heap, no panics). Platform crates define `Interrupt` enum mapping names to vector positions. `extern "interrupt"` is the low-level form.

- [ ] **Critical sections.** Implement `critical_section.acquire()` returning an RAII `CriticalSectionGuard` that disables/re-enables interrupts. Guard type is `#[must_use]`.

- [ ] **C calling convention variants — reserved set.** `extern "C"`, `"C-unwind"`, `"interrupt"` are implemented; `"stdcall"`, `"fastcall"`, `"win64"`, `"sysv64"` are reserved (parse + targeted "reserved at v1" diagnostic, no codegen). *(Roadmap § Phase 11 Embedded row; no prior tracker entry — authored 2026-06-06.)*

- [ ] **`float_in_serialized_type` lint.** Warn when `#[derive(Serialize)]` / `#[derive(Deserialize)]` contains an `f32`/`f64` field — JSON has no NaN encoding, format consumers follow IEEE. Suppressible per-field with `#[allow(float_in_serialized_type)]`. Lands alongside the Serialize/Deserialize derives (G32 in `phase-8-stdlib-floor.md`), post-v1. *(Roadmap § Phase 11 Embedded row; no prior tracker entry — authored 2026-06-06.)*

### Codegen Optimization (IR quality pass)

**Goal:** reduce the Phase 7 ≤2x gap to ≤10% of equivalent hand-written Rust on compute-bound benchmarks. Ships at the end of v1 because IR-quality polish only pays off once the long-tail stdlib is the last thing being measured. Foundation already landed: the LLVM `default<O2>` mid-end pipeline (`phase-7-codegen.md`, ✓ 2026-05-10, commit `280ce2d`) — these entries are the deferred follow-ups that entry named, plus the metadata-emission work that makes O2 bite harder. *(Entries authored 2026-06-06 from `roadmap.md § Phase 11 > Codegen Optimization`; previously roadmap-only.)*

- [ ] **Compiler-driven inline hints.** Emit `alwaysinline` / `noinline` LLVM attributes from call-site analysis (callee size, call frequency in loops, single-call-site detection). Distinct from the *user-facing* `#[inline]` / `#[cold]` attribute surface tracked at `phase-8-stdlib-floor.md § Pre-existing Phase 8 work > Codegen hint attributes` (Phase 5.2/7) — this pass must compose with it (user hints win over heuristics).

- [ ] **Alias metadata.** Emit `noalias` on owned and `mut ref` parameters — Kāra's declared parameter modes are an aliasing oracle the optimizer currently never sees — plus `tbaa` type-based alias-analysis tags on loads/stores. Likely the highest-leverage row in this section for loop-heavy kernels (enables LICM / vectorization that O2 currently can't prove safe).

- [ ] **Arithmetic flags.** Emit `nsw`/`nuw` on integer ops where overflow is defined-UB in Kāra semantics, unlocking strength-reduction and induction-variable optimizations.

- [ ] **LTO in `karac build --release`.** Link-time optimization across the user-code module and the runtime static archive. Named as a deferred follow-up on the O2 entry in `phase-7-codegen.md` ("LTO across the runtime + user-code modules — separate slice; runtime ships as separate static archive"). Must respect the lean/full/wasm three-archive resolution scheme (`CLAUDE.md § Three archives`) — LTO applies against the resolved archive, not a merged superset.

- [ ] **Static branch hints from effect analysis (`llvm.expect` emission).** Emit the `llvm.expect` intrinsic on branch conditions where effect analysis can predict likelihood (e.g., error-path branches behind `panics`-effect calls, `Err` arms of `?` desugaring). **This is not PGO** — no instrumentation, no profile collection, no recompile loop. Real PGO (instrumented + AutoFDO) is deferred to post-v1; see [`deferred.md § Profile-Guided Optimization Loop`](../deferred.md#profile-guided-optimization-loop).

- [ ] **Re-bench the kata corpus after each row lands.** Per the corpus-wide-win discipline: every cross-cutting codegen change re-runs the kata bench pipeline (seq lane first), diffs old→new per metric, and classifies each delta (noise / load / improvement / regression) before the row is closed.
