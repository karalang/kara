- [x] **Effect-checker: target-provided-resource-set parameterization.** *(2026-06-05)* `src/effectchecker/target_gate.rs` (E0411), run after inference settles for the current target (`native` until `--target` lands; the per-target build matrix loops over this same pass). Key decisions, all from `design.md § Cross-target Compilation`:
  - **Reachability-rooted**: gate walks the transitive call graph from `main` ("a function in the transitive call graph of a native entry point") — unreachable declarations don't fire (they're dead code). Library builds / test entry points: follow-on noted below.
  - **Provider discharge**: the design's primary SSR pattern binds per-target providers over target-foreign resources (`Display => HtmlStringBuilder` on native), so a `providers { }` binding discharges the resource for the subtree. Effect inference is provider-transparent, so the gate recovers bindings via synthetic `__providers_bind::<R>` pseudo-calls recorded by the inference walk (the `__modbind_*` precedent; real consumers filter/miss harmlessly) and carries a discharged-set down the DFS. Granularity is the FUNCTION (a binding anywhere in `f` discharges for all of `f`) — slightly lenient for mixed bodies, documented in the module header.
  - **Direct-use attribution**: a violation fires at the function whose effect has `Direct` origin OR `Callee` origin naming a body-less callee (host fn / extern / builtin — not graph nodes, charged at the call site). Body-having callees are examined at their own node where discharge state may differ.
  - **Alias canonicalization** (the prerequisite recorded here): renamed gated-stdlib clones carry `EffectResourceDecl::canonical_host_name` (set by the single-file expansion's alias rename); the gate canonicalizes before keying the table — `writes(Screen)` gates as Display. User resources are exempt (no host identity; they exist wherever a provider exists). A user-declared `effect resource Display;` claims the host identity by string — consistent with the `std.wasi` redeclaration philosophy.
  - **Table**: `target::{is_host_resource, target_provides}`; design.md's table gained a `ProcessTable` row (native-only — doc gap found while transcribing).
  - Diagnostic: ``target `native` does not provide resource 'Display' — required on the call chain main → helper → paint; …`` with actionable fixes (gate with `#[target]`, bind a provider, keep off entry path). Tests: effectchecker ×6 (chain, unreachable, SSR discharge, user exempt, native-provided pass, sibling-path fires), cli ×2 (alias-evasion check-path, SSR pass).

- [ ] **`karac run` and effect violations — decide warn vs. abort.** Surfaced by the target-gate slice: `cmd_run` never invokes `pipeline.effectcheck()` at all — the lenient script path treats typecheck errors as warnings and effect violations as completely invisible (a program rejected by `karac check` for an undeclared effect, or for a target-gate violation, executes silently under `karac run`). Source-level semantics match native (function appears to block on receive). Verify no user-observable difference from native except scheduling. Applies to both sequential-default lowering and `--features wasm-threads` lowering from Q5 — the yield-to-event-loop behavior is orthogonal to whether the underlying scheduling is single-threaded or multi-worker.

- [ ] **`std.web.time.after(duration) -> Channel[()]` and event-stream wrappers.** Ship the library-layer wrappers that convert callback-driven host APIs into channel producers: `time.after(duration)` (setTimeout), `time.every(duration)` (setInterval), `events.clicks(element)` / `events.keydown(element)` / etc. (DOM events), `time.animation_frames()` (requestAnimationFrame). Each returns a `Channel[T]` with a library-managed sender wired to the underlying `host fn` callback registration. Lifetime: closing the channel deregisters the host listener.

- [ ] **WASM entry-point discovery.** A public function is exported in a WASM build when it is `pub` AND (it is target-tagged `#[target(wasm_browser)]` / `#[target(wasm_wasi)]` matching the build target, OR it is target-agnostic and transitively reachable from a target-tagged entry point). Standard dead-code elimination applies. No separate `#[export]` attribute. For `--bindings browser`, verify exported parameter/return types satisfy the `host fn` boundary restriction (primitives / `Copy` / opaque handles); emit a diagnostic on violation. For `--bindings component`, permit richer WIT-expressible types.

- [ ] **SSR provider-injection pattern — documentation and example.** Ship a worked SSR example in `docs/book/` (or `examples/`) showing the target-agnostic-component + per-target-provider pattern from `design.md § Cross-target Compilation`. The purpose is to keep users away from `#[target(...)]` in component bodies; the attribute is for entry points and irreducible forks only. Example should cover one component that renders to HTML on server and to DOM on client, with both entry points calling the same component function.

- [ ] **GPU routing decision — `#[gpu]` annotation design.** Decide how code is split between CPU and GPU. Recommendation: explicit `#[gpu]` attribute on functions, compiler verifies GPU-safety via effects (no `reads`, `writes`, `sends`, `receives`, `allocates(Heap)`, `panics`, no recursion). Layout groups (`group physics { position, velocity }`) map to GPU buffers. User declares intent, compiler enforces the rules. Design options: (A) fully automatic — too much magic, (B) explicit `#[gpu]` — recommended, (C) layout-driven — could complement B.
