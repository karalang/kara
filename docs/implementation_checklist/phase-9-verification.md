- [ ] **Contracts (`requires`/`ensures`/`invariant`).** Implement contract annotations. `requires`/`ensures` on functions, `invariant` on struct fields. Runtime-checked in debug builds, stripped in release. Declaration ordering: signature ŌåÆ effects ŌåÆ requires ŌåÆ ensures ŌåÆ where ŌåÆ body. See design.md ŌĆö "Contract evaluation semantics" for the four rules below. Ordered implementation steps:
  1. [ ] **Parse `old(expr)` as a special form inside `ensures` clauses.** Reject `old(...)` occurrences in `requires`, `invariant`, and outside contracts entirely. Reject `old(result)` with a pointed diagnostic.
  2. [ ] **Contract-expression effect-set check (`ŌŖå {panics}`).** After effect inference on the contract expression, verify the inferred set is a subset of `{panics}`. On violation, emit a compile error pointing at the offending call with message naming the forbidden effect (e.g., `"reads(FileSystem) not permitted in contract expression"`). The `panics` effect is permitted because indexing/division/unwrap are idiomatic in predicates; no other effect is permitted.
  3. [ ] **`old(expr)` desugaring with `Clone` requirement.** For each `old(expr)` occurrence, synthesize a function-entry local that clones `expr` before the body runs. Rewrite the postcondition to read from the local. If the type of `expr` does not implement `Clone`, emit a compile error with message `"\`old(...)\` requires the captured expression to be Clone"`.
  4. [ ] **Consumed-parameter reference check in ensures.** When an `ensures` clause syntactically references a parameter that was consumed (e.g., a bare-`self` receiver or an owned parameter whose value is moved into the return), emit a compile error suggesting `old(self)` or `old(self.field)`. Detection is a scope check at ensures-expression resolution time.
  5. [ ] **Invariant insertion at every `pub` method exit.** For each type with an `invariant` block, insert an invariant check immediately before every return point in every `pub` method (including constructors ŌĆö pub associated functions that return `Self`). The rule is structural: applies regardless of receiver mode or whether the method syntactically mutates. Private methods do not check. Applies uniformly to `struct`, `shared struct`, and `shared enum`.
  5b. [ ] **`impl invariant` ŌĆö all-method invariant scope.** Extend the parser, AST, and typechecker to support `impl invariant <expr>` alongside the plain `invariant <expr>` form inside struct bodies.
