# Brainstorming v49 — `Env.var()` error type (`VarError`) shape and placement

**Status:** ✓ RESOLVED — 2026-04-23. `VarError` is a single-variant enum (`NotPresent`, no payload) registered under `std.env` via the `register_builtin_types` shim. Q1 → B, Q2 → B, Q3 → A. Q4 informed Q1 and is not a separate decision.

## Disposition

| Question | Resolution |
|---|---|
| Q1 — variant set | **B** — single `NotPresent` variant, no payload |
| Q2 — type placement | **B** — lives at `std.env.VarError`, kept out of `PRELUDE_TYPES` / `PRELUDE_VARIANTS` |
| Q3 — shim vs. wait for real stdlib | **A** — registered in `register_builtin_types` now alongside `Option`/`Result`/`Ordering` |
| Q4 — does interpreter need `NotUnicode` payload? | No — Kāra's strict-UTF-8 `String` has no carrier for the offending bytes; this is the structural argument that closes Q1 = A |
| `Result[String, VarError]` vs `Option[String]` (raised inline) | Keep `Result` — preserves forward compatibility for future variants (e.g. permission errors) without a breaking change |

## Why

1. **Q1 = B (single variant).** Rust's `VarError::NotUnicode(OsString)` has no Kāra analog — `String` is strictly UTF-8, and introducing `Vec[u8]` as a payload type for one error case widens the surface for thin gain. Callers almost always `unwrap_or` on env vars; the UTF-8 distinction is rarely actionable. Single-variant enum (rather than unit type) preserves room to add reasons later without a breaking change.

2. **Q2 = B (stdlib module, not prelude).** Prelude expansion is the direction of least reversibility. Existing `PRELUDE_TYPES` are general-purpose (`Option`, `Result`, `Vec`, …); `VarError` is one-method-specific. Friction is lower than it looks — callers don't name the error type for `env.var("X").unwrap_or(...)`; only pattern arms naming `VarError.NotPresent` need an import.

3. **Q3 = A (shim now).** Same placeholder treatment as `Option`/`Result`/`Ordering`. Waiting for `runtime/stdlib/*.kara` materialisation would block `Env.var()` indefinitely. Migrates alongside the other prelude/stdlib types when real stdlib lands.

## Known incongruity (until std.env synthetic module exists)

Today the typechecker keys its enum table by simple name only (`env.enums.get("VarError")`), so the shim treatment leaks the simple name `VarError` into all scopes — the user can write `Result[String, VarError]` or `match` on `VarError.NotPresent` without an import. Strict Q2 = B would require a synthetic `std.env` module (parallel to `std.prelude`) plus an actual `import std.env.VarError;` lookup path. That's tracked as part of the same stdlib-materialisation CR that replaces `register_builtin_types`. No new TODO needed.

The Q2 = B intent is honoured by *not* adding `VarError` to `PRELUDE_TYPES` or `NotPresent` to `PRELUDE_VARIANTS` — when the synthetic-module work happens, the type stops being globally visible and the simple name will require `import std.env.VarError;`. Code written today against the leak will need an import added at that point; the migration is mechanical (one import per file).

## Original framing (for record)

`docs/design.md:2799` specifies:

```
fn env.var(name: ref String) -> Result[String, VarError] with reads(Env)
```

`VarError` is named in the return type but is **not defined anywhere**:

- Not in `PRELUDE_TYPES` (`src/prelude.rs:67`)
- Not spec'd as an enum or struct in `docs/design.md`
- Not mentioned in `docs/roadmap.md` or `docs/implementation_checklist.md`
- No precedent in the stdlib shim in `src/typechecker.rs::register_builtin_types`

The interpreter's ambient-default slice for `Env` could not land `Env.var()` cleanly until this was settled. `Env.args()` (also at line 2799) does not have this dependency and landed alongside this doc opening.

### Prior-art reference

Rust's `std::env::VarError`:

```rust
pub enum VarError {
    NotPresent,
    NotUnicode(OsString),
}
```

Two variants, second carries the offending non-UTF-8 bytes so callers can report them. Kāra has no `OsString` analog, which closes the door on the second variant for us (Q4).

### Open questions (now resolved — see Disposition)

- **Q1 — Variant set.** A: two variants matching Rust. **B (chosen): single `NotPresent`.** C: opaque struct.
- **Q2 — Type placement.** A: prelude. **B (chosen): stdlib module `std.env.VarError`.** C: bare anonymous unit type.
- **Q3 — Shim vs. wait for real stdlib.** **A (chosen): shim now in `register_builtin_types`.** B: defer `Env.var` until real stdlib lands.
- **Q4 — Does interpreter need the `NotUnicode` payload at all?** No (Kāra's `String` is strictly UTF-8) — informs Q1 independently of lean.
